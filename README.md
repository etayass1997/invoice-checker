# בדיקת התאמת חשבוניות להזמנות

## מטרת הכלי

הכלי מבצע השוואה אוטומטית בין דוח הזמנות לבין דוח חשבוניות במטרה לוודא שסכומי ההזמנות תואמים לסכומי החשבוניות שהופקו בפועל.

הבדיקה מיועדת לבקרת תשלומי עמלות, כאשר עמלה משולמת רק עבור הזמנות שעזבו את המלון ושילמו בפועל (Checkout).

---

## קבצי קלט

### קובץ הזמנות
דוח הזמנות מאתר הבית או ממערכת ההזמנות.

עמודות חובה:

- c_master_id
- c_reservation_id
- c_reservation_status
- c_price_amount

### קובץ חשבוניות
דוח חשבוניות מאופטימה.

עמודות חובה:

- c_folio_number
- invoice_amount

---

## אופן הפעולה

1. המערכת מסננת רק הזמנות בסטטוס Checkout.
2. לכל הזמנה נוצר מפתח התאמה ייחודי.
3. לכל חשבונית נוצר מפתח התאמה מקביל.
4. המערכת מבצעת התאמה בין ההזמנות לחשבוניות.
5. מחושב סכום ההזמנה כולל מע"מ.
6. מתבצעת השוואה מול סכום החשבונית.
7. מופק דוח Excel מפורט הכולל סיכום וחריגות.

---

## תוצאות הבדיקה

### תקין
נמצאה חשבונית תואמת וההפרש בין הסכומים נמצא בתחום הסף שהוגדר.

### הפרש סכומים – לבדיקה
נמצאה התאמה בין ההזמנה לחשבונית אך קיים פער כספי הדורש בדיקה.

### הזמנת Checkout ללא חשבונית
הזמנה שסומנה כ-Checkout אך לא נמצאה עבורה חשבונית.

### חשבונית ללא הזמנה תואמת
נמצאה חשבונית שלא נמצאה עבורה הזמנה מתאימה.

### לא Checkout
הזמנות שלא הגיעו לסטטוס Checkout ולכן אינן נכללות בחישוב העמלות.

---

## מבנה דוח האקסל

הקובץ המופק כולל את הלשוניות הבאות:

2. תקינים
3. חריגים
4. חסר חשבונית
5. חשבונית ללא הזמנה
6. לא Checkout
